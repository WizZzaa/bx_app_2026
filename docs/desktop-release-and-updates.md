# ПК-релиз BX и бесшовные обновления

## Что получает пользователь

- Первый раз пользователь скачивает `BX-win32-x64-Setup.exe` со страницы релиза.
- Установщик Squirrel ставит BX в профиль Windows без ручной распаковки архива.
- После установки BX проверяет новые опубликованные версии через `update.electronjs.org`.
- Новая версия загружается в фоне как Squirrel-пакет `.nupkg`.
- BX показывает уведомление и предлагает перезапуск. Работу пользователя принудительно не прерываем.
- Если пользователь выбирает «позже», загруженная версия применяется при следующем запуске.

## Почему релиз собирает GitHub Actions

Windows-установщик должен собираться на Windows runner. Workflow сначала выполняет typecheck, lint и тесты, затем на чистом `windows-latest` создаёт и публикует:

- `BX-win32-x64-Setup.exe` — первичная установка и мост со старых версий;
- `app-<version>-full.nupkg` — пакет автоматического обновления;
- `RELEASES` — индекс Squirrel-пакетов.

Windows ZIP намеренно не публикуется: старые версии BX ищут ассет по `win32-x64`, поэтому должны выбрать настоящий установщик, а не архив.

## Правила выпуска версии

1. Поднять версию в `package.json` и добавить запись в `src/shared/version.ts`.
2. Выполнить `npm run premake`.
3. Создать тег той же версии, например `v2.29.1`, и отправить его в GitHub.
4. Дождаться зелёного workflow `Release`.
5. Проверить, что GitHub Release опубликован и содержит Setup.exe, `.nupkg` и `RELEASES`.
6. На Windows проверить два сценария: чистая установка и обновление с предыдущей версии.

Теги должны быть корректным SemVer и релизы не должны быть draft/prerelease: публичный сервис обновлений такие релизы не раздаёт.

## Совместимость и безопасность

- Squirrel package id оставлен `app`, чтобы не разорвать цепочку уже установленных версий.
- Пользовательские данные хранятся отдельно от каталога программы и не удаляются при обновлении.
- Приложение не перезапускается самовольно посреди работы.
- Для массового распространения потребуется Windows code-signing сертификат: без него установка и обновление работают, но SmartScreen может показывать предупреждение неизвестного издателя.
- Для бесшовного обновления macOS потребуется подпись Developer ID и notarization. До этого macOS использует ручной fallback релиза.
- Linux лучше обновлять через собственный репозиторий пакетов; встроенный Electron autoUpdater его не поддерживает.
